새로 배운 내용
Alt키를 누른 채 같은 단어를 한꺼번에 여러 개 선택해서 다른 단어로 한 번에 바꿀 수 있는 기능을 처음 알게 되었는데, 
앞으로 유용하게 많이 쓸 수 있을 것 같다. Visual Studio Code에서 터미널을 사용할 수 있고, 또 그걸 통해서
 Github에 파일을 업로드 할 수 있다는 사실도 알게 되었다.  Github의 사용이 항상 어렵게 느껴졌었는데, 
강의 내용에 나온대로 따라해보니, 이제는 적어도 '도대체  Github는 어떨 때 쓰는 건가'하는 생각은 들지 않는다.

참고할 만한 내용
단축키에 대해서 좀 찾아봤는데 유용한 기능이 많았다. 당장은 익숙하지 않아서 많이 사용할 것 같지는 않지만,
익숙해지면 편할 것 같은 기능이 많아 보인다.

Ctrl + Shift + K  -> 해당 커서에 있는 행 삭제
Ctrl + P or E -> Quick Open(파일명 여기서 입력하면 한번에 찾을 수 있음)
Ctrl + U -> 셀렉션 되돌리기
Ctrl + Shift + \ -> () 또는 {} 괄호의 시작점이나 끝점으로 이동함
Ctrl + F2  -> 커서가 위치한 동일한 모든 단어를 선택

회고
강의 시간에 간단하게라도, 수업 때 배우는 내용의 역사라든지, 재미있는 사실을 알려주셔서 앞으로 배우게 될
내용에 조금 더 흥미가 생기는 것 같다. 이번주는 과제를 하면서 어디를 개선하면 좋을지 생각하는 것이 어려웠다. 
어떤 점을 개선해야 더 나은 페이지가 되는지 생각을 해본 적이 많이 없기 때문인 것 같다. 앞으로 수업을 들을 때는
실습을 하면서 에러가 없이 잘 실행되는지만 생각하지 말고, 다른 코드로 어떻게 바뀔 수 있는지, 
그렇게 바꾸면 어떤 점이 좋은지 더 생각하면서 수업을 들어야겠다는 생각을 했다.
세 가지를 변경했는데, 첫번째로 emp_insert_process를 변경해서 원래 insert 페이지로 돌아가게 되어 있던 것을
emp_select로 돌아가게 변경하영 추가된 정보를 바로 확인할 수 있게 하였다.
두번째로는 emp_insert에서 모든 타입이 text로 되어 있던 것으 항목에 알맞게 변경했다.
직원 번호는 number, 생일과 채용 날짜는 date, 성별은 보기를 주어서 그 중에 선택할 수 있게 바꾸었다.
세번째로, header 함수를 사용하여 별도의 클릭 과정 없이 index 페이지와 emp_select 페이지로 돌아가게 변경하였다.
