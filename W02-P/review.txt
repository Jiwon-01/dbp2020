새로 배운 내용
1. 우선 깃허브를 한 번도 사용해 본 적이 없어서 굉장히 새로웠다. 익숙하지 않은 환경이라 파일 몇 개 올리는 것도 어려워서, 구글에서 이것 저것 찾아보면서 했다.
2. InnoDB라는 것도 처음 들어봤다. 수업 때 교수님께서 간략하게 설명해주셨지만 어떤 개념인지 더 알고 싶어서 관련 내용을 더 찾아봤다.  
InnoDB는 대용량 데이터를 처리할 때 최대의 퍼포먼스를 내도록 설계되었기 때문에 높은 퍼포먼스가 필요한 대용량 사이트에 적합하다고 한다.
트랜잭션 관리, 복구가 필요한 경우나 정렬 등의 구문이 들어가는 경우 InnoDB를 사용하는 것이 유리하다.
3. AUTO_INCREMENT 속성을 처음 알게 되었다. 이 속성으로 INSERT문을 사용할 때마다 자동으로 num 값이 증가하게 저장할 수 있다.
1부터 100까지의 데이터가 들어가 있을 때, 50부터 100의 데이터를 삭제하였다고 하면, InnoDB에서 새로 INSERT 했을 때 num 값은 101이 아닌 51이 된다.
MyISAM에서는 101이 된다.

문제 발생 내용
강의를 들으면서 같이 실습을 하고 있던 중, CREATE 버튼을 누르면 submit 버튼을 누르지 않아도 자동으로 데이터가 topic 테이블로 INSERT 되는 것을 모르고 있다가,
나중에 topic 테이블을 데이터베이스에 연결시켜보니 데이터가 너무 많이 들어가 있어서 필요하지 않은 데이터를 지워야 하는 상황이 있었다.

회고
html문을 배웠던 것이 조금씩 기억나면서 코드를 이해하는 데 도움이 되었다. 네트워크나 서버에 관해 배우는 수업을 들어본 적이 없어서
많은 내용이 생소하고 어려운 점도 있었다. 데이터베이스 수업이나 데이터베이스 설계 수업과는 확실히 다른 내용을 배우다 보니
낯설고 어렵게 느껴지기는 하지만 중간에 포기하지 않고 끝까지 수업을 잘 따라가고 싶은 마음이다.
